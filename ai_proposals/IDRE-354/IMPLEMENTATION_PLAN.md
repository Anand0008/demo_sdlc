## IDRE-354: Refining Admin Closure Process

**Jira Ticket:** [IDRE-354](https://orchidsoftware.atlassian.net//browse/IDRE-354)

## Summary
Refine the Admin Closure process by moving specific closure actions (Withdrawal, Settlement, Other, and early Resubmission) to the main Case Card for all users, while restricting the Admin Closure dropdown to Dismissal and late Resubmission for Team Leads and above. Implement stage-aware financial logic for fee retention and refunding based on the selected closure reason.

## Implementation Plan

**Step 1: Add Closure Actions to Case Card**  
Update the Case Card UI to include new action buttons/dropdown items for 'Resubmission' (only in Initial Eligibility), 'Withdrawal', 'Settlement', and 'Other'. These should be visible to all roles. Ensure that clicking these actions opens a modal requiring a mandatory text note before saving.
Files: `components/case-detail-view.tsx`

**Step 2: Refine Admin Closure Dropdown**  
Modify the existing 'Admin Closure' dropdown to only show 'Dismissal' and 'Resubmission'. Restrict the visibility of this dropdown to Team Leads and above, and only show it when the case status is past Initial Eligibility (e.g., Pending Payments, Final Eligibility). Require a mandatory text note for these actions.
Files: `components/case-detail-view.tsx`

**Step 3: Implement Financial and Status Logic for Closures**  
Implement or update the server action handling case closures to apply the correct financial logic based on the selected reason and current case stage. For Initial Eligibility: Resubmission (100% Admin fee), Withdrawal/Settlement (50% Entity, 100% Admin), Other (100% Entity and Admin). For Post-Initial Eligibility: Dismissal/Resubmission (Refund 100% Entity, Retain 100% Admin), Withdrawal/Settlement (Retain 50% Entity, 100% Admin), Other (Retain 100% Entity and Admin). Update the case status accordingly.
Files: `lib/actions/party-case-details.ts`

**Risk Level:** MEDIUM — The changes involve modifying financial logic (fee retention and refunding) and case status transitions, which are critical paths. Incorrect fee calculations could lead to financial discrepancies.
